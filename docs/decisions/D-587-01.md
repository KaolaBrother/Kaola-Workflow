# D-587-01. Freeze-time disjointness-proof hardening

Date: 2026-07-02
Status: Accepted
Issue: #587

## Context

The freeze-time write-set grammar and the cross-node / parallel-group disjointness proof had
three blind spots, each following the same pattern the #381/#388/#425/#431 freeze-wall checks
were built to close: a plan that looks legal — or even disjoint — at freeze silently misbehaves
at runtime, after the point where a cheap freeze-time check could have caught it.

1. **Glob tokens froze clean, then died at the barrier.** A declared write-set token containing a
   glob metacharacter (`**/*.md`, `src/*.js`) parsed in-grammar and could even look disjoint from
   a sibling leg's write set — `areaForPath` degenerates a glob token to a bogus coarse area like
   `**`, which trivially doesn't collide with a real path's area. The token never matches anything
   at the **exact-path barrier** (which matches by exact string membership), so the grant is dead:
   the node's actual write dies late at runtime as `write_set_overflow`, the same
   maximally-expensive mid-run consent-halt failure mode the #381/#388 shape checks were built to
   pre-empt.

2. **Cross-node disjointness was case-sensitive on a case-insensitive filesystem.** Two parallel
   legs declaring `Src/x.js` and `src/x.js` passed the freeze-time exact-path and coarse-area
   disjointness checks — `classifier.disjointWriteSets` (declared `fanout(<group>)` members) and
   the inferred antichain-sibling exact-clobber check (#232 concurrent-sibling nodes) — because
   the string comparison was case-exact. On macOS or Windows the two paths are the SAME physical
   file: the plan froze GREEN-disjoint and the two legs would clobber each other's write at
   runtime.

3. **The `.md` allowband is invisible to both the disjointness proof and the barrier.**
   `CHANGELOG.md` / `README.md` / `docs/**` are barrier-invisible by design (#424) — a node may
   write there without declaring it in its write set, and the per-node barrier never flags it.
   That is safe for a serial plan, but for a parallel group (a declared fan-out or an inferred
   antichain-sibling pair) it means two concurrently-open legs can both silently write the
   allowband — invisible to the freeze disjointness proof (different exact paths, or an
   undeclared path, can look disjoint) AND to the runtime barrier — and `git merge` silently
   both-applies the edits, producing a corrupted or contradictory doc/changelog merge with no
   refusal anywhere in the pipeline.

## Decision

- **`glob_in_path` (freeze-only).** `validatePlan`'s per-token write-set shape loop refuses a
  token containing a glob metacharacter (`* ? [ ] { }`), checked in the same pass as the existing
  backslash / directory-shaped / `..` shape refusals (after backslash, before directory-shaped).
  Authoring fix: expand the glob to the concrete files it stands for.
- **Cross-node case-fold (unconditional).** `classifier.disjointWriteSets` and the freeze-time
  antichain-sibling exact-clobber check now lowercase the path — and, for the coarse-area arm, the
  area — before comparing across nodes/legs. This is deliberately narrow: `classifier.normalizeRepoPath`
  itself is NOT case-folded (a global fold would corrupt display and error strings, and would
  break the case-exact per-node barrier match), and the existing same-node sibling `case_collision`
  check (#388) is untouched — the fold is added only at the cross-node compare. Over-blocking on a
  case-sensitive Linux filesystem (refusing a plan that would actually be safe there) is accepted
  as the safe direction, consistent with the existing write-set hygiene philosophy of erring
  toward refusal.
- **`parallel_allowband_collision` (freeze-only).** A parallel group — a declared
  `fanout(<group>)` group or an inferred antichain-sibling pair — refuses when 2 or more legs each
  declare an allowband surface (`CHANGELOG.md`, `README.md`, or any `docs/**` path), even when the
  declared surfaces differ (e.g. `CHANGELOG.md` on one leg, `README.md` on another). The
  `kaola-workflow/{project}/**` workflow-state band is excluded from this rule — per-node `.cache`
  evidence legitimately differs per leg, and folding it in would false-refuse every parallel plan.
  Authoring fix: declare the shared docs/changelog write on exactly one leg (typically a
  designated synthesizer or doc-updater leg) and leave the other legs silent on it.

All three checks are mirrored byte-identically across the `scripts/`, Codex (`plugins/kaola-workflow`),
GitLab, and Gitea plan-validator/classifier copies.

## Consequences

- A plan authoring a glob write-set token, a case-variant write-set collision across parallel
  legs, or a multi-leg allowband write now refuses at the cheap freeze-time gate instead of
  reaching a mid-run consent halt or a silent runtime clobber.
- Serial (non-parallel) plans are unaffected by the allowband rule — it is scoped to declared
  fan-out groups and inferred antichain-sibling pairs only.
- The `--parallel-safe` runtime co-open check (`plan-validator.js --parallel-safe`) retains its
  own separate exact-file-overlap loop, which this change does NOT case-fold; its coarse/shared-infra
  arm is case-folded incidentally because it delegates to `classifier.disjointWriteSets`. Aligning
  the two loops for full symmetry is out of scope here.
- No existing refusal is relaxed; all three additions are new refusals layered onto previously
  in-grammar plans, so a plan that does not trigger one of the three new checks remains in-grammar
  and hash-stable.

## Extended by D-702-01 (#702)

The `parallel_allowband_collision` rule above was **surface-granular** — it treated the entire
`docs/**` band as one collision surface and refused ≥2 legs each declaring any allowband path.
D-702-01 makes it **file-granular**: the arm now splits HARD (`CHANGELOG.md` / `README.md` / a
glob or dir-shaped `docs/` token) from SOFT (an exact-file `docs/**` token). The HARD refusal is
unchanged (still catches `CHANGELOG.md` on one leg + `README.md` on another). SOFT collisions —
exact-file-disjoint docs siblings under a post-dominating `code-reviewer` — are now ADMITTED, so
the natural "N independent docs writers in parallel" shape freezes in-grammar and co-opens as
isolated legs. The runtime half is a leg-scoped per-leg-barrier visibility extension (docs become
`write_set_overflow`-visible inside a leg) plus `README.md` joining the PROTECTED indexes. The
glob/dir and case-fold hardening above is untouched.
