# D-629-01. Close three cross-edition guard-net blind spots: hooks.json family parity, agents.toml byte-identity, and edition-sync create-on-missing

Date: 2026-07-08
Status: Accepted
Issue: #629 (primary — the cross-edition guard-net blind spots)
Related: #418.1 (the existing `CONFIG_HOOKS_FAMILY`/`normalizeConfigHooks()` normalization pattern
         this decision extends to `hooks/hooks.json`); #422/D-422-01 (the existing
         `BYTE_IDENTICAL_GROUPS` pattern this decision extends to the `config/agents.toml` triple);
         #307 (the four-chain cross-edition gate this guard net backs)

## Context

Three blind spots existed in the cross-edition guard net enforced by `scripts/validate-script-sync.js`
and `scripts/edition-sync.js`:

1. `hooks/hooks.json` (root + gitlab + gitea) was explicitly EXCLUDED from parity checking on a stale
   rationale, even though its only real per-forge divergence is the same shape
   `CONFIG_HOOKS_FAMILY`/`normalizeConfigHooks()` already handles for `config/hooks.json` — a
   forge-renamed `kaola-workflow-compact-context` script-name token.
2. The `plugins/kaola-workflow{,-gitlab,-gitea}/config/agents.toml` triple was byte-identical at HEAD
   but had no `BYTE_IDENTICAL_GROUPS` entry — only derived NAME parity was checked, so a byte
   divergence between the three copies could land undetected.
3. `scripts/edition-sync.js` `runWrite()` steps (b) (codex-sync of `COMMON_SCRIPTS`) and (c)
   (byte-identical-group sync) only wrote a mirror when the target already EXISTED
   (`fs.existsSync(...) && ...`). A newly-enrolled member whose mirror was absent was silently
   skipped, even though `validate-script-sync.js` reds "Missing files" for exactly that member — the
   documented `sync:editions` remedy was a dead end for the enrollment workflow: running `--write` did
   not fix the red it was supposed to fix.

## Decision

1. Add a `HOOKS_JSON_FAMILY` (root + gitlab + gitea `hooks/hooks.json`) checked via a new
   `normalizeHooksJson()` that rewrites only the `kaola-workflow-compact-context` token to its
   forge-renamed form — the verified sole per-forge diff.
2. Add the `config/agents.toml` triple as a `BYTE_IDENTICAL_GROUPS` entry.
3. Extract two shared primitives — `checkNormalizedFamily(family, normalizeFn, rootDir,
   normalizedKind)` and `checkByteIdenticalGroup(group, rootDir)` — as behavior-preserving refactors
   of the pre-existing inline loops (used by the new families AND the existing
   `RENAME_NORMALIZED_FAMILIES`/`CONFIG_HOOKS_FAMILY`/pre-existing byte groups), verified line-by-line
   to reproduce the old missing/drift semantics exactly.
4. Fix `edition-sync.js`: extract `syncIfDrift(rootDir, rel, content)` (write when the target is
   ABSENT or its content differs, matching step (a)'s pre-existing create-on-missing pattern) and
   rewire `runWrite()` steps (b) and (c) to use it, dropping the `existsSync` gate that previously
   skipped an absent mirror entirely.

All three fixes were RED-first: bullets 1–2 proven via a stashed pre-fix `validate-script-sync.js`
(existence + mechanism assertions failing structurally — 5 failures / 31 passed); bullet 3 via a
synthetic tmp-dir-enrolled member with an absent mirror (`TypeError: syncIfDrift is not a function`
pre-impl; post-fix, the mirror is created and idempotency/drift-overwrite regressions are guarded).
No `hooks.json`/`agents.toml` data file was rewritten by this change — both new families are already
in parity at HEAD; the create-on-missing fix was proven only against a synthetic fixture, never the
real repo tree.

## Consequences

- The enrollment loop (`validate-script-sync.js` reds "Missing files" → run `edition-sync.js --write`
  → chain goes green) is now fail-closed AND self-healing for `COMMON_SCRIPTS` and
  `BYTE_IDENTICAL_GROUPS` members: `--write` creates the missing mirror instead of silently no-op'ing.
- `validate-script-sync.js` now reports 26 byte-identical groups (up from 25) and 2 hooks.json
  families (root `config/hooks.json`'s `CONFIG_HOOKS_FAMILY` plus the new `hooks/hooks.json`
  `HOOKS_JSON_FAMILY`).
- The two shared-primitive extractions are pure refactors — every pre-existing family's check
  behavior (missing-reference handling, drift message strings, Buffer-vs-utf8 comparison) is
  unchanged; only the terminal summary line's wording changed.
- Cross-edition #307: this diff touches `scripts/edition-sync.js`, `scripts/validate-script-sync.js`,
  and their test files; all four `npm run test:kaola-workflow:{claude,codex,gitlab,gitea}` chains
  were run sequentially and are green.

## Non-goals / accepted residual

- `edition-sync.js --check` (`runCheck`) still covers only `GENERATED_AGGREGATORS` — it stays green
  on a missing `COMMON_SCRIPTS`/byte-group mirror that `--write` would now fix. This is a pre-existing
  asymmetry, not introduced or worsened by this decision: `validate-script-sync.js` covers those
  families in-chain, so the enrollment loop is fail-closed end-to-end even though `--check` alone is
  not the complete story. A filed follow-up tracks widening `--check` to cover the same universe as
  `--write`.
- No new normalization rule beyond the single sanctioned compact-context token: an OVER-normalization
  probe (renaming an unrelated `.sh` hook token to its forge name) was verified to still red, proving
  `normalizeHooksJson()` cannot mask a non-sanctioned divergence.

## Alternatives considered

- **Widen `edition-sync.js --check` in this same change.** Rejected: `--check`'s scope
  (`GENERATED_AGGREGATORS` only) is an existing, separately-scoped concern; conflating its widening
  with this fix's already-multi-part diff risked scope creep for a non-blocking residual. Filed as a
  follow-up instead.
- **Byte-compare `hooks/hooks.json` directly instead of normalizing.** Rejected: the three copies
  differ by design in exactly one sanctioned per-forge token (proven by diffing root↔gitea at HEAD:
  the only real difference is that single compact-context command line), so a plain byte-compare
  would permanently red on a difference that is not a defect — the same reasoning that motivated
  `CONFIG_HOOKS_FAMILY`'s normalization for `config/hooks.json`.
