# D-582-02. Codex tiered dispatch requires fresh child-session effort proof

**Date:** 2026-07-01
**Status:** Accepted
**Issue:** #582
**Related:** #451, #581, #584, D-582-01

## Context

D-582-01 moved Codex tier selection onto descriptor-level `reasoning_effort` and treated V2
task-name dispatch as the intended effective path. Post-merge installed-runtime validation on this
machine refuted that assumption for the `sonnet -> high` tier:

- Codex CLI: `0.142.5`.
- Installed plugin cache was refreshed from the merged repo and matched source.
- Installed-cache doctor reported user scope `dispatch_mode: "v2-task-name"` and
  `multi_agent_v2_enabled: true`.
- A V2 child spawned with `fork_turns: "none"` and `reasoning_effort: "high"` recorded
  `turn_context.effort: "xhigh"` in its child session JSONL.
- Repeating the high probe with explicit `model: "gpt-5.5"` still recorded `xhigh`.
- The installed role profile did not pin `model_reasoning_effort`; the child inherited the
  parent/global `xhigh` effort.

That means config, feature state, and parent-side spawn descriptors are not enough evidence that a
non-null Codex tier was honored.

## Decision

- `dispatchEffort()` continues to emit the descriptor mapping:
  `opus -> xhigh`, `sonnet -> high`, absent/blank -> inherit.
- A non-null `dispatch.codex_reasoning_effort` is dispatchable only after a fresh child-session
  effort proof exists for the exact requested effort.
- The proof must inspect the spawned child session JSONL `turn_context.effort`.
- Proof from config text, `codex features list`, parent-side descriptors, spawn arguments, or child
  self-report is insufficient.
- The proof requirement applies to both V2 task-name dispatch and V1 fallback.
- If proof is absent, stale, or failing, the executor refuses before dispatch with
  `codex_effort_override_unavailable`.
- Absent/blank model tiers remain the only intentional Codex effort inheritance path.

## Consequences

- V2 remains the identity path (`task_name`) but is not itself an effort proof.
- Current local Codex sessions with global `model_reasoning_effort = "xhigh"` must not claim
  `sonnet -> high` effort integrity until a child JSONL records `effort: "high"`.
- The six plan-run command/skill surfaces and validators pin this fail-closed rule so future
  guidance cannot silently regress to trusting descriptor text alone.
