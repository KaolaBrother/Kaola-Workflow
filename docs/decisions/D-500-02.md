## Context

D-500-01 (existing) surfaced three parallel-write makespan levers — L1 (`write_overlap_policy`
relaxation), L2 (`KAOLA_LEG_ISOLATION` activation recipe), and L3 (`speculative_open_policy:consent`
prose) — that were inert on the live path, with a per-lever WIRE vs. RELABEL recommendation. The owner
approved the build run on 2026-06-16; this record captures that acceptance and the implementation
shipped as the #500 build.

## Decision

The owner accepted all three levers as WIRE and the build run implemented them. The accepted direction
per lever:

- **L1 (write_overlap_policy relaxation):** WIRE, leg-COUPLED safe way (the guarded path recommended
  by D-500-01 (existing), not the cheap unsafe consent-forward).
- **L2 (KAOLA_LEG_ISOLATION):** WIRE as the honesty fix — the complete activation recipe documented
  across all six #400 plan-run surfaces; stale "DORMANT" code comments corrected to reflect the
  shipped state.
- **L3 (speculative_open_policy:consent):** WIRE the prose — card, six surface markers, and a driver
  line; planner rubric filed as a separate follow-up.

## Implementation (shipped in #500 build)

### L1 — write_overlap_policy relaxation (leg-coupled)

`tryFormLaneGroup` in `adaptive-node.js` was updated to forward `--write-overlap-consent` only when
`resolveLegIsolation(process.env) && opts.writeOverlapConsent` are BOTH true. This closes the
#283/#303 clobber window: a shared-infra co-open can no longer form unless legs are fully provisioned
(containment on + consent explicit). A new shared-infra-coarse end-to-end test in
`test-adaptive-node.js` exercises the full relaxation → form → provision → synthesize → barrier path
and was adversarially mutation-verified to be non-vacuous.

### L2 — KAOLA_LEG_ISOLATION activation recipe (honesty fix)

The complete activation recipe — `KAOLA_LANE_CONTAINMENT` + `KAOLA_LEG_ISOLATION` +
`--write-overlap-consent` + `write_overlap_policy:coarse` — was documented across the six #400
plan-run surfaces (3 Claude commands + 3 Codex SKILL packs). The stale "DORMANT" code comments in
`adaptive-node.js` at the affected lines were reworded to reference ADR-0010 with the framing
"containment, not construction," accurately reflecting the shipped per-leg mechanism.

### L3 — speculative_open_policy:consent prose wire

A new `docs/plan-run-cards/speculative-open.md` card was minted, six `<!-- CARD: speculative-open -->`
markers placed across the six plan-run surfaces, and a driver line added to make the consent path
reachable and honestly documented. The realizable proactive win (a planner rubric teaching the
post-gate read-only topology) is filed as a separate follow-up (#513) and was not implied by this
prose wire.

## Cross-edition reach and verification

All four editions of `adaptive-node.js` were updated via edition-sync. All six #400 plan-run surfaces
received the L2 activation recipe and the L3 speculative-open marker. All four
`npm run test:kaola-workflow:{claude,codex,gitlab,gitea}` chains were run sequentially and green at
finalization. Two LOW cosmetic follow-ups from the code-reviewer are filed as #514.

## Consequences

- **L1:** Shared-infra co-open is now reachable on the live path under the guarded condition
  (leg-isolated + explicit consent). The #283/#303 clobber window is closed by the leg-coupling guard.
- **L2:** The activation recipe for per-leg write parallelism is honestly documented. The "DORMANT"
  label is retired; the feature is described as "containment, not construction" per ADR-0010.
- **L3:** The speculative consent path is reachable and documented. The proactive planner win is
  deferred to #513.
- **Open follow-ups:** #513 (planner rubric for speculative-open proactive use), #514 (cosmetic LOW
  items from code-reviewer).
- **D-500-01 (existing) status:** Remains as the recommendation/investigation record; this record (D-500-02)
  is the acceptance and implementation companion.
