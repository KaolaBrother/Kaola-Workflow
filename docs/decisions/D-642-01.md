# D-642-01. A durable node-to-node information channel: planner-authored `## Node Briefs`, `dispatch.goal_line` / `dispatch.upstream_evidence`, and a close-time consumed-proof

Date: 2026-07-08
Status: Accepted
Issue: #642
Related: #444/D-444-01 (the shared `buildDispatch` producer this channel extends — the pre-built
         but unwired `ctx.goal_line` socket lived there since #444); #591/D-591-01 (the
         `leg_path`/`leg_branch` conditional-attach discipline this channel's fields reuse
         verbatim); #392 (`checkEvidenceShape`, the sibling close gate this decision's consumed-proof
         is deliberately kept consistent with); #643/D-643-01 (the sibling registry/contract
         completion landed in the same run); #641/D-641-01 (the run whose own review/adversary
         gates this run's process mirrors)

## Context

Node-to-node information transfer was not mechanically guaranteed. A producer node's full output —
a `code-architect` blueprint, a `planner` recommendation, a `code-explorer` map, a
`knowledge-lookup` fact set — reached its downstream consumer (often a lower-tier `implementer`/
`tdd-guide`) only through the orchestrator's transient conversational context, never through the
machinery itself. An audit of `adaptive-node.js` found two concrete gaps: `buildDispatch` carried
no upstream pointer at all, not even a bare `depends_on` echo, and `seedEvidenceFile` seeded only a
binding header plus empty token stubs; no routing surface's dispatch instructions told the
orchestrator to relay anything upstream-derived to the consumer. A pre-built `ctx.goal_line` socket
had existed in `buildDispatch` since #444 but was never populated by any caller — the field was
wired for a feature that was never finished.

The risk this channel closes: a compaction, a fresh session, or simply an inattentive orchestrator
mid-run can silently drop the producer's reasoning before it reaches the consumer, and nothing in
the machinery would notice — the consumer's evidence would look complete regardless of whether it
actually incorporated the upstream work.

## Decision

**`## Node Briefs` — a new hash-covered plan section.** One column-0 `### <node-id>` heading per
brief under the `## Node Briefs` h2; the body is the brief text (`goal_line`). Parsed
fence-aware — identically to `classifier.sectionBody`'s own h2/h3 boundary and fence handling, so a
fenced `### x` inside a brief's code block is never mistaken for a header. `computePlanHash`
appends `'\n---BRIEFS---\n' + norm('Node Briefs')` to its hash body **only when** the section is
present (a fence-safe column-0 heading probe, not `sectionBody`, which cannot distinguish absent
from empty) — a briefless plan hashes byte-identically to before, so every existing frozen or
in-flight plan resume-checks unchanged. Freeze gains two typed refusals, both freeze-only (briefs
are hash-covered, so a frozen plan can never carry either defect afterward — nothing to re-check on
`--resume-check`):

- `brief_unknown_node` — a brief names an id absent from `## Nodes`.
- `brief_duplicate_node` — two `### <node-id>` blocks reuse the same id. The parser resolves
  duplicates first-wins and the dispatch channel silently uses only the first, so without this
  refusal a planner authoring error (a copy-paste duplicate id) would vanish with no signal at all —
  asymmetric with the freeze wall's existing duplicate-`## Nodes`-id check. Surfaced by this run's own
  `code-reviewer` as a non-blocking nit and fixed in the same run rather than filed as a follow-up.

**`buildDispatch` gains two conditionally-attached fields**, following the exact `leg_path`
discipline (#591): `goal_line` (already a socket; now populated from the node's brief when one
exists) and a new `upstream_evidence: [{node_id, role, path}]` array, derived from the node's
`depends_on` — re-looked-up from the frozen plan content at every call site (never carried from a
next-action result), so it can never drift from what was actually frozen. Only entries whose
upstream role is a PRODUCER role (`code-architect`, `code-explorer`, `knowledge-lookup`, `planner`,
`issue-scout`, `synthesizer`) are included; `path` is the project-qualified
`kaola-workflow/<project>/.cache/<id>.md` evidence file (the same barrier-exempt band
`record-evidence`/`--verify` already use). Both fields are wired at all three openers
(`open-next`, `open-ready`, and `close-and-open-next`'s fused advance); a batch `open-ready` open
derives each member's own brief and own upstream list individually, never shared across members. A
briefless plan or a root/depless node produces a `dispatch` byte-identical to pre-#642 (neither key
attached) — `seedEvidenceFile` additionally appends one empty `upstream_read: <up-id>` stub line
per producer upstream for an `IMPLEMENT_ROLES` consumer, mirroring its existing `forceRotate`
re-seed path so a reopen re-emits the stub.

**Close-time consumed-proof (`checkUpstreamConsumed`), reusing the evidence-binding nonce as an
anti-fabrication read-proof.** The dispatch card carries `{node_id, role, path}` — **never** a
nonce. A consumer proves it actually opened the upstream file by echoing a column-0
`upstream_read: <up-id> <nonce>` line matching that upstream's CURRENT line-1
`evidence-binding: <up-id> <nonce>` header. Run in both close paths
(`close-and-open-next`/`close-node`) immediately after the evidence-shape check and before the
barrier — zero ledger mutation on refuse. Scope and exemptions, evaluated in order: a root node (no
deps) or a non-producer upstream requires nothing; an upstream whose ledger status is `n/a`
(skipped) is exempt (its own evidence proves nothing, so neither does a missing echo of it); a
producer upstream whose `upstream_read: <up-id>` KEY is absent from the consumer's evidence is
exempt (DD-5 back-compat, below); otherwise a missing or mismatched echo is a typed
`{ result: 'refuse', reason: 'upstream_not_consumed', offending: <up-id>, expectedPath }` — **HARD**
(the close genuinely refuses, zero mutation) only when the consumer's own role is an
`IMPLEMENT_ROLES` member (`tdd-guide`, `build-error-resolver`, `implementer`, `metric-optimizer`);
every other pairing (e.g. a gate depending on a producer) is advisory only and never blocks.
Reopening the UPSTREAM (`reopen-node`, `forceRotate`) rotates its line-1 nonce, so any consumer
that already echoed the old nonce fails this check on its own next close until it re-reads and
re-echoes; reopening the CONSUMER discards its own prior echo the same way (its evidence is
re-seeded).

**A universal n/a-skip carve-out** — `if (content.trim().startsWith('n/a')) return out;` —
mirrors `checkEvidenceShape`'s own long-standing n/a carve-out (every non-gate role, including
every `IMPLEMENT_ROLES` member, may legitimately skip its work by recording `n/a — <reason>`
evidence). Before this carve-out, an `IMPLEMENT_ROLES` consumer legitimately skipped (e.g. a
`build-error-resolver` skipped because the build was already green) still carried the seeded/
re-injected `upstream_read: <up-id>` key with no echo, so it HARD-refused — the shape gate and the
consumed gate disagreed on the identical evidence, and the only in-run recovery was echoing a
nonce into a node that did no work, a semantically false read-proof the machinery itself would
have demanded, inverting the anti-fabrication intent it exists to serve. This run's own
`code-reviewer` gate found the gap (empirically reproduced against the live worktree before the
fix) and it was fixed in the same run: a skipped consumer proves nothing, so the two close gates
now render the same verdict on the same n/a evidence.

**Anti-fabrication invariant, verified end-to-end.** The upstream nonce is emitted from exactly one
place — line 1 of the upstream's own evidence file — and nowhere else: not the dispatch card, not
any of the three openers, not `deriveDispatchChannel` (plan content only, never touches `.cache`),
not the seed or re-injection (which write the bare key, no nonce), and not the cached
`.cache/<op>-envelope.json`. This run's `adversarial-verifier` gate ran a live-CLI disproof attempt
against real scratch repos: grepped the full serialized `open-next` envelope, the fused-advance
envelope, `orient --json`, and the cached summary envelope for the upstream's known nonce — absent
from all of them in every case; attempted omission (ride the back-compat exemption), a wrong nonce,
echoing the consumer's own dispatched nonce, and a smuggled/indented echo — every attempt refused
with zero mutation; the happy path (reading the upstream's real line-1 nonce and echoing it)
closed cleanly.

**Resume re-hydration.** No new opener code beyond the dispatch fields — the `--summary` open path
already caches the full result envelope at `.cache/<subcommand>-envelope.json`. Once the dispatch
card carries `goal_line` + `upstream_evidence`, that cache IS the durable re-dispatch context: a
fresh-shell `orient --json` finds the in-progress node in the ledger, and the routing prose (the
skeleton + the `resume.md` card) points the orchestrator at the cached envelope as authoritative —
never reconstruct the brief or the upstream pointers from a prior turn's transcript.

## Consequences

- A frozen plan MAY author `## Node Briefs`; a plan that doesn't is completely unaffected (byte-
  identical hash, dispatch card, and seed behavior to before this change).
- The planner-facing brief syntax is now spelled out in `agents/workflow-planner.md`'s compact-plan
  posture (one column-0 `### <node-id>` heading per brief; any other layout — bullets, a table, bold
  names — parses as zero briefs) so a malformed section fails loudly rather than silently no-opping
  the entire channel on its first real use — a gap this run's own `code-reviewer` surfaced as a
  blocking finding and this run fixed in place.
- An `IMPLEMENT_ROLES` node with ≥1 producer upstream must now genuinely read and echo that
  upstream's evidence before it can close; skipping the read is a hard, zero-mutation refuse — the
  only way past it is to actually open the file.
- `kaola-workflow-adaptive-node.js` and `kaola-workflow-plan-validator.js` are GENERATED
  aggregators; the canonical edits were regenerated to the codex byte mirror and both forge rename
  ports via `npm run sync:editions` (byte-/rename-identical — no hand-porting).
- Tests (claude-only surfaces): `test-adaptive-handoff.js` pins `brief_unknown_node` and
  `brief_duplicate_node` end-to-end plus the post-freeze brief-edit hash-tamper case;
  `test-adaptive-node.js` and `simulate-workflow-walkthrough.js` pin card emission at all three
  openers (byte-identical when briefless/root), the consumed-proof happy path, each negative
  (missing echo, stale echo via reopen rotation, wrong echo, smuggled echo), the n/a-skip carve-out,
  the advisory (non-`IMPLEMENT_ROLES`) path, the fabrication-resistance grep, and resume
  re-derivation from the cached envelope. All four
  `npm run test:kaola-workflow:{claude,codex,gitlab,gitea}` chains verified green (run
  sequentially).

## Non-goals

- Does not change the shape or grammar of any existing plan section other than adding the new
  optional `## Node Briefs` h2.
- Does not require every plan to author briefs, and does not change any behavior for a plan that
  doesn't — this is strictly additive.
- Does not weaken `checkEvidenceShape` or any other existing close gate; the consumed-proof is a
  new, independent check layered before the barrier, not a replacement for the shape check.
