## Context

Disjoint-write parallelism had been DESIGNED, wired, and proven safe across the D-419/D-500 series,
yet remained effectively unreachable on the live path. To co-open a planner-proven-disjoint
(`parallel_safe`) write antichain as isolated parallel legs, an operator had to satisfy a
**four-way conjunction**: `KAOLA_LANE_CONTAINMENT` on **and** `KAOLA_LEG_ISOLATION` on **and**
`--write-overlap-consent` passed **and** `write_overlap_policy:coarse`. Every clause defaulted OFF
(D-419-02 Part 2 shipped the capability "wired but inert"; D-500-02 L1 relaxed `write_overlap_policy`
only in the leg-COUPLED-safe way, still behind the full recipe).

The practical consequence: **almost every `implement` frontier ran serial.** A planner could prove a
write antichain genuinely disjoint at freeze time (`plan-validator.js` already classifies every
antichain write pair — exact-file overlap ⟹ refuse, coarse/shared-infra ⟹ ask, truly disjoint ⟹
pass), and the runtime would still hard-gate every write node to `write_node_exclusive` unless an
operator had pre-configured the full toggle stack. The toggle was **over-cautious**: it withheld a
safe, proven default behind manual ceremony that no ordinary run performed, sacrificing the largest
remaining makespan lever (CLAUDE.md:37) for a guard the correctness net already provides structurally.

This conflicts with the design precedence (CLAUDE.md): accuracy is non-negotiable (#1), but **then**
automation & efficiency (#2) — and parallelism is a means, to be taken at the highest *safe*
concurrency when the work genuinely decomposes (#472). A default that forces decomposed, proven-safe
work serial spends makespan a smaller approach would not.

## Decision

**Planner-proven-disjoint (`parallel_safe`) write frontiers co-open as isolated parallel legs BY
DEFAULT — no operator toggle.** The four-way activation conjunction is retired as the gate for the
*disjoint* case.

Concretely:

- **New default predicate.** `kaola-workflow-adaptive-schema.js` (×4 byte-identical drift anchor)
  adds `PARALLEL_WRITES_ENV = 'KAOLA_PARALLEL_WRITES'` and `parallelWritesDefaultOn(env)` — default
  **TRUE**; only `'0'` / `'false'` / `'no'` flip it off.
- **Co-open keys on the default, not the recipe.** In `kaola-workflow-adaptive-node.js`,
  `legCoupled = parallelWritesDefaultOn(process.env)` (was the `resolveLegIsolation && consent`
  conjunction). The co-open gate drops its `containment &&` precondition — it now fires on
  `if (legCoupled && writeNodes.length >= 2)`. Close-path member detection keys on
  `running0.lane_group` (dropping the `resolveLaneContainment` read); the orphan sweep is gated on
  `parallelWritesDefaultOn(process.env)`.
- **Legs + synthesizer are the correctness net.** Each co-opened disjoint leg is contained in its
  own per-leg git worktree (#463 / ADR-0010), and the mandatory synthesizer reconcile is the join.
  Containment is provided by the *mechanism* (isolated legs + reconcile), not by an operator
  remembering to flip a flag. This is precisely the reintroduction condition ADR-0008 set for write
  isolation — now satisfied by default rather than by recipe.
- **Overlap stays consent-gated.** Only genuinely-overlapping (non-disjoint) write frontiers remain
  behind explicit consent — `--write-overlap-consent` **and** `write_overlap_policy != off`. The
  default-on flip does **not** relax the overlap case; the planner's disjointness classification is
  the discriminator (disjoint ⟹ default co-open; coarse/shared-infra overlap ⟹ consent; exact-file
  overlap ⟹ refuse, unchanged).
- **`KAOLA_PARALLEL_WRITES=0` is the opt-out.** A host that cannot isolate legs, or an operator who
  wants the serial floor, sets the env to `0` (or `false`/`no`) and every write frontier degrades to
  the serial `write_node_exclusive` path — the byte-equivalent pre-flip behavior.
- **Env-default activation across the runtimes × forges.** The default is active out of the box at
  install for the three runtimes (Claude, Codex, opencode) × the three forges (GitHub, GitLab,
  Gitea) — no per-edition special-case lane; the shared schema constant carries the default
  byte-identically across all four chains.

## The #498 invariant is preserved

Co-open **always** provisions legs: `groupForm ⟺ legCoupled ⟺ legs provisioned`. The default-on
flip changes *when* `legCoupled` is true (now: the disjoint-default predicate, not the recipe
conjunction), but it never decouples group-formation from leg-provisioning. There is no path where a
co-open forms WITHOUT isolated legs — the legless, attribution-blind union barrier is never taken.
Every disjoint co-open is leg-contained and synthesizer-reconciled, or it does not co-open at all.

## Alternatives considered

- **Keep the four-way toggle, document it better (status quo + honesty fix).** This is what D-500-02
  L2 already did — and the empirical outcome is that ordinary runs still went serial, because the
  default was OFF and no run flipped four flags. Documenting an OFF default does not make a proven-safe
  behavior reachable. REJECTED: the toggle was the problem, not the documentation.
- **Default-on for ALL writes (including overlap), consent only on exact-file refuse.** REJECTED:
  violates precedence #1. A coarse/shared-infra overlap has a real clobber window (#283/#303) that the
  per-leg worktrees + reconcile do not by themselves resolve; the planner's `ask` classification for
  that case is load-bearing. Default-on is correct **only** for the planner-proven-disjoint set.
- **A new operator opt-IN flag (e.g. `--parallel-writes`).** REJECTED on the same grounds as #538's
  switch-axis flip: an opt-in that no ordinary run sets reproduces exactly the unreachability we are
  fixing. The safe behavior must be the default; the toggle exists only to opt *out*.

## Consequences

**Positive:**
- The largest remaining makespan lever (CLAUDE.md:37) is taken by default: a faithfully-decomposed,
  planner-proven-disjoint write frontier now co-opens at width instead of degrading to a serial chain.
- No operator ceremony. The proven-safe behavior runs out of the box; the agent has no toggle stack to
  resolve and the planner's disjointness proof is honored automatically.
- Correctness is unchanged. The per-leg worktree containment + mandatory synthesizer reconcile are the
  same net D-419/D-500 proved; the overlap consent gate and the exact-file refuse are untouched. The
  #498 co-open ⟹ legs invariant holds.

**Negative / note:**
- A host that cannot provision per-leg worktrees (no git history, offline, or a constrained
  environment) relies on the `KAOLA_PARALLEL_WRITES=0` opt-out to fall back to serial; the default
  assumes leg isolation is available. The provisioning preconditions are the same as the existing
  worktree-native path.
- The advanced/legacy `KAOLA_LANE_CONTAINMENT` / `KAOLA_LEG_ISOLATION` knobs no longer gate the
  disjoint default — they remain as defense-in-depth / advanced controls. The write-lane hook
  (`hooks/kaola-workflow-write-lane.sh`) is **deliberately unchanged**: per-leg git-worktree isolation
  is the real containment; the hook is defense-in-depth only.
- D-419-01 / D-419-02 Part 2 / D-500-02 L1/L2 are superseded on the default-posture axis only. Their
  kernel model, per-leg mechanism, emit-envelope reason taxonomy, and overlap-consent semantics all
  stand; only the activation default flips from recipe-gated-OFF to disjoint-default-ON.
