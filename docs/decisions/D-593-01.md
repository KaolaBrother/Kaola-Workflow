# D-593-01. Relax the `coarse` write co-open class BY DEFAULT under the retained safety net

Date: 2026-07-02
Status: Accepted
Issue: #593
Related: `docs/decisions/D-546-01.md` (titled D-546-G2; the prior default-relax of the
             `shared-infra` class under the same retained net — this decision extends the
             identical reasoning to `coarse`); D-419-01 (Part 1, the coordination kernel and the
             `write_overlap_policy` field this decision makes vestigial); D-587-01 (the
             freeze-time disjointness hygiene — glob/case-fold/allowband — this decision's
             `hasUnresolvableEntry` guard reuses the same glob/directory-shape detection)

## Context

`writeOverlapRelaxable` (`kaola-workflow-plan-validator.js`) is the single predicate that
decides whether an overlapping write pair may be downgraded from a `red`/`yellow`
disjointness verdict to `ok` (co-openable). Two retained-safety-net invariants gate EVERY
relaxation at EVERY class and tier: **NET-1**, a `synthesizer`/`code-reviewer` gate
post-dominates the relaxed legs themselves (a leg-scoped `gateUncovered` check, not a
whole-plan check — a whole-plan check would be vacuously-empty for a docs-only leg and
relax an unreviewed frontier); **NET-2**, neither leg's declared write set contains a
PROTECTED concrete file.

Before this decision, `writeOverlapRelaxable` split three ways by `classifier.disjointWriteSets`'s
`kind`:

- **`shared-infra`** (both legs' declared files sit in the same `SHARED_INFRA` area — e.g.
  two `scripts/` files — but are exact-file-disjoint) already relaxed BY DEFAULT under the
  retained net, no policy or consent required (the `#546-G2` decision).
- **`coarse`** (both legs share a NON-shared top-level area — e.g. two forge-port files both
  under `plugins/` — but are exact-file-disjoint) relaxed ONLY when the operator opted in
  with `write_overlap_policy ∈ {disjoint, coarse}` in the frozen plan's `## Meta` AND the
  per-run, never-persisted `--write-overlap-consent` flag was passed at `open-ready` time.
- **`exact`** (the same path, or a case-collision) never relaxed at this seam.

This asymmetry had a concrete cost in THIS repo. Every cross-edition write frontier — the
canonical `plugins/kaola-workflow/scripts/`, `plugins/kaola-workflow-gitlab/scripts/`, and
`plugins/kaola-workflow-gitea/scripts/` forge ports — shares the single top-level area
`plugins/`. A planner-authored antichain of two forge-port-writing legs is exact-file-disjoint
by construction (each leg writes its own forge's files) but ALWAYS classified `coarse`, so it
ALWAYS serial-degraded unless an operator manually supplied `--write-overlap-consent` on
every single `open-ready` call for that frontier — authored parallelism that could never
actually run concurrently by default, in exactly the shape of write frontier this repo
authors constantly.

The `coarse` class's own safety argument is IDENTICAL to `shared-infra`'s, once exact-file
disjointness is established: the sets are disjoint by construction (no path in common), the
gate (NET-1) covers the merged union at review time, and the per-leg commit barrier still
catches any TEXTUAL conflict a leg introduces regardless of the freeze-time classification.
The operator-consent ceremony that gated `coarse` added nothing the structural net does not
already guarantee — the same conclusion `#546-G2` reached for `shared-infra`.

One genuine gap existed that `shared-infra`'s default-relax did not need to consider as
directly: `disjointWriteSets` classifies by STRING comparison of declared write-set entries.
A directory-shaped token (a trailing `/`) or a glob (`* ? [ ] { }`) can compare
string-distinct from a sibling's entries while still covering files the sibling leg also
writes — exact-path disjointness is UNPROVABLE for such an entry, so relaxing on the strength
of a string comparison alone would be unsound.

## Decision

**`writeOverlapRelaxable`'s `coarse` arm now relaxes BY DEFAULT under the SAME retained net
(NET-1 + NET-2) as `shared-infra`**, subject to one additional guard:

```
if (dj.kind === 'coarse') {
  if (hasUnresolvableEntry(setA) || hasUnresolvableEntry(setB)) return false;
  return true;
}
```

- **`hasUnresolvableEntry(set)`** — new pure-string helper: an entry is NOT exactly-resolvable
  (and so the pair is NOT provably disjoint) iff it ends in `/` (directory-shaped) or contains
  a glob metacharacter (`* ? [ ] { }`). When either leg's set carries such an entry, `coarse`
  keeps today's PREVENT verdict for that pair — the same shapes the freeze-wall glob/directory
  detection already refuses at freeze (D-587-01), so this guard only bites a legacy in-flight
  plan frozen by a pre-freeze-wall validator.
- **`exact` NEVER relaxes here** — unchanged. Same path, or a case-collision (caught by the
  classifier's case-fold, D-587-01), stays blocking at every tier; no flag, policy, or
  consent bypasses it. Real reconciliation of a genuine textual overlap remains the runtime
  `merge_conflict` barrier's job (deferred).
- **`write_overlap_policy` and `--write-overlap-consent` become VESTIGIAL at this seam.**
  Both stay PARSED — a frozen plan authored under the old contract, or an operator habit of
  passing `--write-overlap-consent`, continues to work exactly as before — but neither value
  changes the outcome for `coarse` anymore: `policy:off` with no consent now relaxes
  byte-identically to `policy:coarse` with consent. Neither field is repurposed as a
  net-bypass, and `write_overlap_policy: exact` is deliberately NOT squatted (it stays
  reserved for the still-deferred exact-file optimism axis).
- **Classifier verdict purity is unchanged.** `classifier.disjointWriteSets` still emits
  `red` / `kind: 'coarse'` for this case — the downgrade lives ONLY at the
  `writeOverlapRelaxable` callsite in `plan-validator.js`. Every verdict-only reader
  (`scanClaimedOverlap`, the G-SEL-4 select-arm check, the freeze-time #232 concurrent-sibling
  antichain loop) is unaffected, so freeze-time behavior (the antichain "must ask" surfacing
  a genuinely-shared-area pair to the operator BEFORE the plan is frozen) is unchanged.

## Consequences

- Every planner-authored write frontier whose legs are exact-file-disjoint — regardless of
  whether they share a top-level area — now co-opens as an isolated-leg lane group BY
  DEFAULT, with no operator action required, PROVIDED the retained net holds. This directly
  unblocks this repo's own recurring cross-edition-port write shape (multiple forge ports
  under `plugins/`), which previously could never co-open without manual per-run consent.
- `--write-overlap-consent` and `write_overlap_policy` stop doing anything observable for
  `coarse` (both classes it used to gate — `shared-infra` and now `coarse` — relax
  unconditionally under the net). They remain parsed for frozen-plan back-compat; a future
  decision may retire them outright once no in-flight frozen plan still references them, but
  that is out of this decision's scope.
- The `hasUnresolvableEntry` resolvability guard is the only new safety surface introduced.
  It is pure string logic with no filesystem access, mirroring the freeze-wall glob/directory
  detection so the two stay in agreement.
- `docs/api.md`, `docs/architecture.md`, and the six plan-run routing surfaces (3 Claude
  commands + 3 Codex SKILL packs) — which previously documented `coarse` as requiring
  `--write-overlap-consent` — are updated to the default-relax contract in the same change.
- Cross-edition (#307): `kaola-workflow-plan-validator.js` is a GENERATED_AGGREGATOR
  (canonical + Codex byte-twin + GitLab/Gitea rename-generated forge ports); all four
  `npm run test:kaola-workflow:{claude,codex,gitlab,gitea}` chains must stay green. A
  pre-existing pair of `test-commit-node.js` floor assertions (`T463-FLOOR-consent` /
  `T463-FLOOR-off`) that pinned the OLD consent-gated `coarse` behavior were re-pinned to the
  new default-relax floor (asserting `ok`/`relaxed[kind:'coarse']`, not merely a non-refusal)
  — their sibling floors (`gate-absent`, `docs-only-gate`, `exact`, `PROTECTED`) are
  unaffected and remain unchanged.

## Alternatives considered

- **Keep `coarse` consent-gated (status quo).** Rejected: this is the exact defect the
  issue names — a genuinely-safe, exact-file-disjoint write frontier that happens to share a
  top-level area serial-degrades by default with no path to authored parallelism short of an
  operator manually consenting on every run. The safety argument for `coarse` under the
  retained net is identical to `shared-infra`'s, which already defaults ON; leaving `coarse`
  behind was an inconsistency, not a deliberate extra safeguard.
- **Relax `coarse` unconditionally, with no additional net (plain green).** Rejected: this
  would also relax a `coarse` pair carrying a directory-shaped or glob entry whose exact-path
  disjointness is unprovable by string comparison alone — a real (if narrow, since the
  freeze-wall already refuses these shapes going forward) soundness gap. The
  `hasUnresolvableEntry` guard closes it at negligible cost (pure string check, no new I/O).
- **Squat `write_overlap_policy: exact` or repurpose `--write-overlap-consent` as the
  resolvability override.** Rejected: `exact` is reserved for a distinct, still-deferred
  design axis (exact-file optimism / real-time reconciliation of a genuine textual overlap),
  and repurposing the consent flag as a bypass for the resolvability guard would reopen the
  exact soundness gap this decision closes — an operator could then wave through an
  unprovable pair. The resolvability guard is unconditional by design.
- **Retire `write_overlap_policy`/`--write-overlap-consent` outright in this same change.**
  Rejected as out of scope: both fields are hash-covered plan content / a documented CLI
  flag respectively, and retiring them would be a breaking change to any already-frozen plan
  or operator script that still passes them. Keeping them parsed-but-vestigial is the
  non-breaking choice; retirement is deferred to a dedicated follow-up if ever pursued.

## Extended by D-702-01 (#702)

This decision relaxed the `coarse` class at the RUNTIME `--parallel-safe` seam only; the
FREEZE-time declared-`fanout()`-group disjointness loop still refused a coarse-RED verdict
unconditionally. D-702-01 LIFTS this same `writeOverlapRelaxable` ladder into the freeze fan-out
loop (via `groupWriteOverlapRelaxable`, a pair-loop reuse — no second comparator), scoped to the
docs allowband: a `fanout()` group whose members declare ONLY exact-file `docs/**` surfaces now
relaxes its coarse-RED verdict at freeze under the SAME retained net (NET-1 gate + NET-2 no-
PROTECTED + `hasUnresolvableEntry` resolvability + exact-never-relaxes). A NON-docs coarse fan-out
keeps today's freeze refusal — the freeze wall stays deliberately stricter than runtime for code.
