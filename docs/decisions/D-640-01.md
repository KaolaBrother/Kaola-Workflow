# D-640-01. Harden the OPT-2 `metric_paths` shape check: refuse absolute-path, backslash, and bare-existing-directory entries

Date: 2026-07-08
Status: Accepted
Issue: #640
Related: #639/D-639-01 (hardened OPT-2 to refuse directory-shaped/glob/`../`-aliasing `metric_paths`
         and explicitly deferred these three residual shapes — bare existing-directory,
         backslash-separated, and absolute-path entries — as a non-blocking defense-in-depth
         follow-up); #634/D-634-01 (the original `metric-optimizer` role and its OPT-1..6 freeze
         rules this decision further hardens)

## Context

#639 tightened OPT-2's `metric_paths` shape check to refuse directory-shaped, glob, and
`../`-aliasing entries — closing the gap where such an entry could cover or alias a write-set file
while comparing string-distinct. Its `## Consequences`/`Related` section explicitly left three
further shape classes unmirrored: a bare existing-directory entry (no trailing slash), a
backslash-separated entry, and an absolute-path entry — all three already refused on
`declared_write_set` by the write-set freeze-wall (`scripts/kaola-workflow-plan-validator.js`,
~lines 1396-1432) but not yet mirrored onto OPT-2's `metric_paths` check. This residual was filed as
non-blocking defense-in-depth, not fixed at the time. #634's own review/adversary gates, and the
#638-639 bundle's own review/adversary follow-up, both re-surfaced the gap as worth closing rather
than leaving open indefinitely.

The gap is inert today: a backslash-separated path dies at dispatch on a POSIX exact-path barrier
regardless; a bare-existing-directory or absolute `metric_paths` entry depends on the runtime
consumer, and any physical aliasing that slips past is still backstopped by the OPT-5 change-gate
metric reproduction. Closing it now is pure completeness — a defense-in-depth hardening, not a fix
for an observed failure.

## Decision

Mirror the write-set freeze-wall's three additional shape refusals onto OPT-2's `metric_paths`
check, in `shapeReason(p)` (`scripts/kaola-workflow-plan-validator.js`, ~line 1537):

- **absolute path** — a `/`-prefix (Unix) or Windows drive-letter (`C:`) token → reason token
  `absolute_path`.
- **backslash-separated** — `p.includes('\\')` → reason token `backslash_in_path`.
- **bare existing-directory** — a slash-less token with no trailing slash that
  `fs.statSync(path.join(optRoot, p)).isDirectory()` resolves to a real directory (a `statSync` throw
  means the path does not exist yet — a legitimate not-yet-created file — and is a clean skip)
  → reason token `bare-existing-directory`.

These reuse the freeze-wall's exact check forms rather than a hand-rolled reimplementation, keeping
the two call sites structurally identical. Precedence matches the freeze-wall: the existing
directory-shaped/glob/`../`-aliasing check runs first, then absolute-path, then backslash, then
bare-existing-directory — so a Windows path like `C:\foo` reports `absolute_path`, not
`backslash_in_path`, the same order the freeze-wall already uses. The freeze-wall's own `freezeRoot`
is block-scoped to the write-set loop and out of scope at the OPT-2 site, so a local
`optRoot = opts.root || process.cwd()` is recomputed there. All refusals stay prefixed `OPT-2:` — no
new `OPT-N` marker, since this closes a residual gap in the same "is the declared metric harness well
-formed and provably isolated" invariant OPT-2 already governs.

## Consequences

- An absolute-path, backslash-separated, or bare-existing-directory `metric_paths` entry now refuses
  at freeze with a typed `OPT-2` marker and a per-path reason tag, instead of silently freezing
  in-grammar as before.
- OPT-2 remains freeze-time-only (no `--resume-check` change) and remains `plan_hash`-covered (the
  `optimize(<id>)` block lives inside `## Meta`, which `computePlanHash` already normalizes in full).
- Regenerated across the codex byte mirror and both forge rename ports via `npm run sync:editions`
  (byte-/rename-identical to the canonical edit — no hand-porting).
- Tests: `simulate-workflow-walkthrough.js`'s `testMetricOptimizerContract` gained RED-first refuse
  fixtures for all three new shapes plus a root-file accept-control fixture (a slash-less path that
  resolves to a real FILE, not a directory, stays green). The existing OPT-1..OPT-6 accept/refuse
  fixtures remain green (no regression). All four
  `npm run test:kaola-workflow:{claude,codex,gitlab,gitea}` chains verified green.
- One cosmetic, non-blocking observation raised independently by both the code-review and
  adversarial-verifier gates: for an entry that is BOTH absolute AND directory-shaped, OPT-2's arm
  order reports the directory-shape reason where the write-set freeze-wall would report
  `absolute_path` (the freeze-wall checks absolute first; OPT-2's directory/glob/`../` arm runs
  first). Refusal behavior is identical either way — the entry refuses either way — so there is no
  accept/refuse divergence between the two checks, only a difference in which reason string is
  reported. Documented here, not fixed, since it changes no observable behavior.

## Non-goals

- Does not add a new `OPT-N` marker — the hardening folds into the existing `OPT-2` marker.
- Does not change the write-set freeze-wall itself (`declared_write_set` checks) — only mirrors its
  existing shape refusals onto `metric_paths`.
