# D-702-01. File-granular allowband co-open for exact-file-disjoint docs write siblings

Date: 2026-07-16
Status: Accepted
Issue: #702
Related: D-587-01 (`parallel_allowband_collision` ‚Äî the surface-granular freeze refusal this
             decision narrows to file-granular); D-593-01 (the `coarse` default-relax under the
             retained net that this decision LIFTS into the freeze fan-out loop and reuses at the
             allowband seam); D-546-01 / D-546-G2 (the `shared-infra` default-relax under the same
             net); D-424-01 (the narrow barrier-invisible allowband band); D-419-01 (the
             coordination kernel + `writeOverlapRelaxable` predicate)

## Context

A consumer adaptive run authored five `doc-updater` writer nodes with pairwise-disjoint,
single-exact-file write sets, all under `docs/domain/knowledge/`. They had no hard content
dependency on one another, yet the planner was forced to author them as a serial chain, costing
~60‚Äì100 minutes of avoidable serialization in one bundle. Any docs-heavy epic pays this on every
run ‚Äî against First Principles axiom 2 ("then save human time") and axiom 3 (parallelism is a means
to use when sufficient).

The block was a **two-blocker** freeze-time wall for the natural "N independent docs writers"
shape:

1. **The coarse-RED disjointness arm.** `docs/` is a top-level path segment, so
   `classifier.areaForPath` returns `docs`, and `docs` is not in `SHARED_INFRA`. Two nodes each
   declaring a different exact `docs/*.md` file therefore get
   `classifier.disjointWriteSets` verdict `{verdict:'red', kind:'coarse'}` ‚Äî a coarse-area
   collision. The runtime `--parallel-safe` check has relaxed exactly this since D-593-01
   (`writeOverlapRelaxable`), but the FREEZE-time declared-`fanout()`-group loop
   (`if (dj.verdict==='red') errors.push(...)`) never called it ‚Äî so an explicitly `fanout()`-tagged
   docs group refused at freeze, before the allowband arm was even reached, and could never freeze
   to reach the runtime relaxation.

2. **The `parallel_allowband_collision` arm (D-587-01).** Even past blocker 1, the freeze proof
   refused when ‚â•2 legs each declared ANY allowband surface (`CHANGELOG.md` / `README.md` /
   `docs/**`). The rule was **surface-granular** ‚Äî it treated the entire `docs/**` band as one
   collision surface ‚Äî on the stated grounds that (a) the per-node barrier never flags allowband
   paths and (b) `git merge` silently both-applies disjoint-region edits.

Both stated risks are already mitigated by machinery the engine ships by default for code paths:
per-leg worktree isolation, the parent-clean fence, a leg-scoped tree-diff (git diff sees `.md`
files fine ‚Äî they are invisible only to the REFUSAL logic), and the synthesizer merge + a
post-dominating gate. For **declared exact-file** allowband writes, per-leg attribution is provable
with the existing leg machinery; the freeze refusal foreclosed parallelism the runtime could
already prove safe.

## Decision

Make the allowband collision proof **file-granular**, split across three seams. Freeze time owns
only the STRUCTURAL admissibility proof (exact-file, pairwise-disjoint, gate-post-dominated, no
HARD surface); runtime owns leg provisioning and ‚Äî when legs are live ‚Äî per-leg attribution.

1. **Freeze ‚Äî lift the `coarse` relaxation into the fan-out loop, scoped to the docs allowband.**
   A `fanout(<group>)` group whose members declare ONLY exact-file `docs/**` allowband surfaces
   (`isAllowbandDocsSurface` on every path of every member) now relaxes its coarse-RED disjointness
   verdict via the SAME `writeOverlapRelaxable` ladder the runtime `--parallel-safe` uses
   (`groupWriteOverlapRelaxable`, a pair-loop reuse of `disjointWriteSets` + `writeOverlapRelaxable`
   ‚Äî no second comparator). NET-1 (a `code-reviewer` post-dominating the group's OWN legs, proved
   with the leg-scoped `gateUncovered`) + NET-2 (no PROTECTED file) + resolvability
   (`hasUnresolvableEntry` ‚Äî glob/dir never relax) + exact-never-relaxes all still hold. A NON-docs
   coarse fan-out (two `api/` or `crates/` legs) keeps today's freeze refusal ‚Äî the freeze wall
   stays deliberately stricter than runtime for code, mirroring the retained `shared-infra`
   "must serialize" wall. The YELLOW shared-infra arm is untouched.

2. **Freeze ‚Äî the allowband arm splits HARD vs SOFT.** A HARD allowband surface ‚Äî a shared
   aggregation index (`CHANGELOG.md` / `README.md`) or a glob/dir-shaped `docs/` token (detected
   via the same `hasUnresolvableEntry` shape check) ‚Äî on ‚â•2 legs still refuses
   `parallel_allowband_collision`, unchanged. This still catches DIFFERENT hard surfaces
   (`CHANGELOG.md` on one leg + `README.md` on another). SOFT surfaces (exact-file `docs/**`) are no
   longer caught here: they are governed by the coarse relaxation in seam 1 (declared fan-out) or,
   for an inferred antichain-sibling pair, admitted per-pair only when proven exact-file-disjoint +
   post-dominating-gate-covered (a pair that also declares non-docs code writes stays a hard refuse).

3. **Runtime ‚Äî the per-leg barrier makes the docs allowband VISIBLE in leg scope.** `barrierCheck`
   threads a new `opts.legScoped` signal, set ONLY by the `--leg-barrier` CLI path. When leg-scoped,
   the barrier's `isExempt` stops exempting `isAllowbandDocsSurface` paths (the narrow
   `docs/**`/`CHANGELOG.md`/`README.md` subset ‚Äî NOT the `kaola-workflow/{project}/**` state band,
   which must stay exempt since per-leg `.cache` evidence legitimately differs). An out-of-declared-
   set docs write that landed in a leg now refuses `write_set_overflow` instead of silently both-
   applying at the synthesis merge. This is UNCONDITIONAL once leg-scoped ‚Äî only a plan that passed
   the fixed freeze grammar can reach this state. The change is keyed strictly on `legScoped`: the
   GROUP/union barrier (`opts.groupMembers`) and the WHOLE-PLAN barrier (no `legScoped`) stay
   byte-identical ‚Äî docs remain exempt there, as they must to back every serial close and finalize
   check.

4. **`README.md` joins the PROTECTED aggregation indexes.** `classifier.PROTECTED_BASENAMES` gains
   `README.md` (alongside `CHANGELOG.md` / `ROADMAP.md`). "Stays single-leg" is thereby a STRUCTURAL
   guarantee via NET-2: the file-granular relaxation refuses any leg group carrying it, at every
   tier, independent of the HARD-surface arm.

## Consequences

- A plan with N exact-file-disjoint `docs/**` write siblings under a post-dominating `code-reviewer`
  now freezes in-grammar (declared `fanout()` group OR inferred antichain pair) and co-opens as
  isolated legs ‚Äî the runtime `--parallel-safe` path already handled this once the plan could reach
  it. Docs-heavy epics stop being forced serial.
- A same-file or glob/dir-shaped docs overlap still refuses (`not pairwise disjoint` /
  `parallel_allowband_collision`); a gate-less docs frontier still refuses at freeze (structural
  NET-1) and serial-degrades at runtime.
- An out-of-declared-set `.md` write inside a leg is caught at the leg barrier
  (`write_set_overflow`) ‚Äî no silent both-apply at merge.
- `CHANGELOG.md` / `README.md` / indexes stay single-leg, enforced BOTH structurally at the freeze
  HARD-surface arm and via NET-2 (PROTECTED), and made barrier-visible in leg scope as a strict,
  harmless improvement.
- Cross-edition (#307): `kaola-workflow-plan-validator.js` and `kaola-workflow-classifier.js` are
  mirrored across the canonical `scripts/`, Codex byte-twin, and GitLab/Gitea forge ports; all four
  `npm run test:kaola-workflow:{claude,codex,gitlab,gitea}` chains must stay green. The mirrors are
  regenerated by `edition-sync.js` from the canonical copies. Test coverage ‚Äî the FIRST for
  `parallel_allowband_collision` beyond the existing walkthrough pins ‚Äî adds freeze regression-pins
  (HARD / glob / gate-uncovered still refuse) and the new relaxed behavior
  (`simulate-workflow-walkthrough.js`), the leg-visible docs-overflow regression + mode-parity
  anti-leak guard (`test-adaptive-node.js`), and the `--parallel-safe` docs floor + `README.md`
  PROTECTED pin (`test-commit-node.js`).

## Alternatives considered

- **Relax ALL coarse fan-out groups at freeze (full parity with runtime).** Rejected: an existing
  invariant (and test) keeps a coarse CODE fan-out ‚Äî two `api/` legs ‚Äî refusing at freeze, the same
  deliberate freeze-stricter-than-runtime posture the `shared-infra` YELLOW wall holds. #702 is
  about the docs allowband specifically, so the freeze relaxation is scoped to pure-docs-allowband
  groups; code coarse overlaps keep today's refusal.
- **Only remove the freeze refusal, leave the leg barrier exempting docs.** Rejected: that would let
  an out-of-declared-set docs write silently both-apply at the synthesis merge ‚Äî the exact hazard
  D-587-01 named. The leg-scoped barrier visibility is the runtime half that makes the freeze
  admission safe.
- **Make docs barrier-visible in ALL barrier modes.** Rejected: the group/union and whole-plan
  barriers back serial closes and the finalize check, where the narrow allowband must stay invisible
  (#424) so a legitimately-undeclared serial docs write is not false-refused. The visibility change
  is keyed strictly on the leg-scoped signal.
- **Add a broader index list to PROTECTED.** Rejected as out of scope: only the named shared
  aggregation surfaces already treated as indexes (`CHANGELOG.md`, `ROADMAP.md`) plus `README.md`
  are protected; inferring a wider list would be speculative.
