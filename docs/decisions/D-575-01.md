# D-575-01. Strip design-rationale provenance from agent-facing prompts

Date: 2026-06-29
Status: Accepted
Issue: #575

## Context

Over 100+ issues of development, the agent-facing prompt surfaces (agent definitions, commands, and
skills across all four editions plus opencode) accumulated a large volume of design-rationale
provenance: inline issue refs (`#NNN`), decision IDs (`D-NNN-NN`), invariant tags (`[INV-NN]`),
ADR citations (`ADR-NNNN` / `ADR NNNN`), and whole prose clauses whose only purpose was to record
the history of a past defect or anti-pattern.

A static scan across 129 files found approximately 636 provenance tokens of these classes. The
accumulation was organic — each issue added a few citations as the author-at-the-time had that
context — but the aggregate effect was:

- **Dispatch-time noise.** Every token is loaded into the agent's context window at run time and
  contributes nothing to the operative rule. The issue number `#472` tells a running agent nothing
  it can act on; the rule it tags is what matters.
- **Invisible aging.** Provenance citations age without any visible decay signal. A reference to
  `#439` cannot self-update when the design evolves; it silently drifts from reality.
- **Mis-placed home.** Provenance has a correct home — `CHANGELOG.md`, `docs/decisions/`, and
  commit messages — where it is both expected and indexed. In prompt surfaces it is orphaned from
  the record it was meant to annotate.

The original scope regex targeted hyphen-form ADR citations (`ADR-NNNN`) but missed the space-form
(`ADR NNNN` e.g. `ADR 0004`). A mid-run investigation node (n4a scope-gap probe) confirmed the gap;
a plan-repair added node n4b to cover the space-form before the strip nodes executed.

## Decision

1. **Strip all provenance from prompt surfaces.** Remove issue refs (`#NNN`), decision IDs
   (`D-NNN-NN`), invariant tags (`[INV-NN]`), and ADR citations (both `ADR-NNNN` and `ADR NNNN`
   forms) from every agent definition, command, and skill file across all four editions
   (claude / codex / gitlab / gitea) and the opencode generated mirrors. Also remove whole defect /
   pattern "clauses" whose only content is historical record of a past anti-pattern, where removal
   leaves no operative rule gap.

2. **No rule meaning changes.** The strip is purely cosmetic with respect to agent behavior. Every
   operative rule stays in place; only its citation history is removed. Reviewers and the
   adversarial-verifier (node n6) gate on semantic + completeness equivalence before and after.

3. **Adopt a durable convention.** Add a concise rule to `CLAUDE.md § Non-Negotiable Rules` and a
   full-coverage section to `docs/conventions.md` that:
   - Names the complete prompt-surface set (agents / commands / skills × all editions + opencode).
   - Lists the four banned token classes and the space-form ADR variant.
   - Defines the allowlist (runtime target-issue variable forms; illustrative user-command examples).
   - Designates the correct provenance homes (CHANGELOG / docs/decisions / commit messages).
   - Notes that machine enforcement is deferred.

## Scope correction — ADR space-form (mid-run plan-repair)

The original strip scope in the frozen plan targeted `ADR-NNNN` (hyphen-form) citations only. Node
n4a (scope-gap probe) found `ADR 0004` (space-form) in multiple plugin agent files that the original
regex passed over. The plan-repair added node n4b (strip ADR space-form) to cover these before
finalization. The banned-token table in `docs/conventions.md` records both forms.

## Verification gates

- **n6 adversarial-verifier** — semantic equivalence check (no operative rule dropped) + completeness
  check (no prompt surface left with banned tokens). Runs after all strip and docs nodes.
- **n7 four-chains** — `npm run test:kaola-workflow:claude && ... :codex && ... :gitlab && ... :gitea`
  (sequentially). The strip touches edition trees across all four, so the cross-edition (#307)
  obligation applies. The opencode generated mirrors are regenerated from canonical and verified by
  `node scripts/test-opencode-edition.js`.

## Consequences

- Prompt surfaces carry only operative rule text. Context-window load from historical citation is
  eliminated.
- Future contributors see the `CLAUDE.md` / `docs/conventions.md` rule before writing new agent
  prose, and have a clear provenance home (docs/decisions, CHANGELOG, commits) to point citations
  toward.
- The provenance record for rules that previously cited `#NNN` inline is preserved in this decision
  file and in the CHANGELOG entry for #575, not lost.

## Deferred

**Lint guard / contract pin.** A durable machine-enforced ban — a carefully allowlisted regex
scanning the prompt-surface file set for banned token patterns — would prevent regression without
relying on review discipline alone. The allowlist requires care: runtime forms (`KAOLA_TARGET_ISSUE=N`,
`"issue N"`) and illustrative user examples (`"work on #42"`) must not be banned. A follow-up issue
should scope and implement this guard.

## Alternatives considered

- **Keep provenance inline, accept the noise.** Rejected: the noise is substantial (636 tokens /
  129 files) and accumulates unboundedly. The correct homes for provenance already exist.
- **Strip only new additions going forward (grandfathering).** Rejected: grandfathering leaves
  the existing 636 tokens permanently, which defeats the goal.
- **Move citations to inline comments rather than removing them.** Rejected: the prompt surfaces are
  Markdown consumed verbatim by agents; there is no comment syntax that survives dispatch. The only
  correct home is outside the prompt surface.
