# D-641-01. Relax the read∥write co-open serialization (G1/G2): leg-contained writer co-open behind live reads + the `observes: scratch` consent tier

Date: 2026-07-08
Status: Accepted
Issue: #641
Related: #622 (relaxed the mirror direction — a READ co-opens alongside a live write lane group,
         with `merge_awaits_read_drain` as the complementary fence; this decision relaxes the WRITE
         direction the same way); #596 (the speculative size-1 lane-group machinery this reuses
         verbatim); #547 / `isValidationInvisible` (the allowband-minus-test-consumed-prose predicate
         the R2 band cap is built on); #644 (the two fenced residuals this run surfaced and deferred)

## Context

A `{read ∥ write}` antichain the planner authored as parallel was serialized by the scheduler: a
mixed frontier hit the read-first-exclusive partition (G1) so only reads opened, then the writer was
held by `write_awaits_drain` (G2) until the reads drained. An audit of the adaptive scheduler's
concurrency-gating guards (19 guards, recorded in #641) classified only **G1** and **G2** as
over-conservative; the other 17 guards were confirmed load-bearing (each protects a crisply-restatable
invariant) and left untouched.

The invariant the G1/G2 serialization protects is **closed-work observation**: a read node —
especially a change-gate — must render its output over a tree that contains only work from *closed*
nodes plus its own scratch. A concurrent writer's in-flight bytes in the observed tree could
false-refute (flag a half-written file) or false-pass (credit in-flight sibling work that later fails
its own barrier). Today that invariant is enforced *temporally* (reads and writes never co-run in the
shared tree). This decision enforces it *structurally* (worktree isolation) where possible, and
*contractually* (a planner-declared observation scope) in one narrow consent-tier case — it is never
dropped.

A key correction the audit made: **barrier-invisible ≠ observation-invisible.** The `docs/**`/CHANGELOG
allowband is exempt from barrier *attribution* (`isBarrierInvisible`) but is fully visible in
`git diff`, and it contains chain-verdict-affecting prose (the #547 test-consumed set:
`CHANGELOG.md`, `README.md`, `docs/api.md`, `docs/workflow-state-contract.md`, `docs/agents-source.md`).
So the safe predicate for a legless co-open is `isValidationInvisible` (allowband **minus**
test-consumed prose), never `isBarrierInvisible`, and a full-diff `code-reviewer` gate is never
co-open-safe against a concurrent shared-tree write.

## Decision

Relax only G1/G2, in two slices, both fail-closed to today's behavior:

- **R1 (default-on, structural).** In the `write_awaits_drain` else-branch of
  `scripts/kaola-workflow-adaptive-node.js`, when the frontier is write-only and every live member is
  a read, attempt a **leg-contained** lane-group co-open (a lone writer forms a size-1 group, reusing
  the #596 speculative-write group machinery verbatim) under exactly four preconditions, each
  fail-closed to today's hold with a typed `serialDegradeReason` label naming the cause: `legCoupled`
  (`parallelWritesDefaultOn`; `KAOLA_PARALLEL_WRITES=0` → `parallel_writes_off`, G12); no live
  `lane_group` (→ `lane_group_live`, G8); parent clean (`parentCarriesProductionDirt` non-pass →
  `parent_dirty`, G5); validator `--parallel-safe` ok across {candidates ∪ live writes} (empty
  selection → `overlaps_live_writer` / `parallel_safe_indeterminate`, G6/G7/G13). The G1 read-first
  partition is unchanged; the **G4 `merge_awaits_read_drain` fence is retained** — the leg's
  last-member merge is still held while any read is live, so the parent tree the reads observe is
  untouched until the drain. The #588 mixed-frontier pin stays for legless writers (G3).

- **R2a (consent-tier, validator freeze contract).** A hash-covered `observes` column on the `## Nodes`
  table. `observes: scratch` freezes only on a read node with role `adversarial-verifier`; the freeze
  validator refuses it on any other role (`observes_scope_role_invalid`), on an unknown scope
  (`observes_scope_unsupported`), and on a node carrying a write set (`observes_scope_not_read`) —
  those roles' definition *is* tree/diff observation, so the annotation would be incoherent. It is
  folded into `computePlanHash`'s normalized nodes body, so `plan_hash` changes when it changes. The
  co-open legality predicate (`isScratchObservableSurface` / `scratchObservableWriteSet`) admits a
  writer whose declared set ⊆ (`isValidationInvisible` allowband — allowband docs **minus** #547
  test-consumed prose) ∪ the writer's own `.cache` evidence file.

- **R2b (consent-tier, adaptive-node consumer).** Over a *dirty* parent (where R1's leg path does not
  apply because a legless writer must see the uncommitted context), a single legless writer co-opens
  behind live reads iff every live read is a freeze-declared `observes: scratch` adversarial-verifier
  and the writer set satisfies the R2a predicate; otherwise the byte-identical `parent_dirty` hold.
  The dispatch card the aggregator composes for the annotated node pins the observation contract
  ("verdict from `.cache` evidence of closed nodes + scratch only; do not read the worktree tree or
  diff"). Annotation absent ⇒ serialize (byte-identical).

Serial-fallback byte-identity is a first-class guarantee (AC5): with `KAOLA_PARALLEL_WRITES=0` + no
running set + ≤1 in_progress, every touched path is byte-identical to before, and no
`serialDegradeReason` field is emitted (mirroring the #616 telemetry pattern — `null` reason means no
field).

## Consequences

- A leg-contained writer now co-opens behind live reads when the four preconditions hold (previously
  always `write_awaits_drain`), and an `observes: scratch`-gated legless docs writer co-opens over a
  dirty parent under the R2 contract. Any precondition miss returns today's hold with a byte-identical
  envelope plus the single additive typed `serialDegradeReason`.
- The 17 must-not-relax guards are untouched, verified byte-for-byte: notably G4
  `merge_awaits_read_drain` (the isolation net), G3 legless exclusivity, G5 parent-dirty, G6/G7/G13
  disjointness, G8 single-lane-group, and G1 read-first partition.
- `scripts/kaola-workflow-adaptive-node.js` and `scripts/kaola-workflow-plan-validator.js` are
  GENERATED aggregators; the canonical edit was regenerated to the codex byte mirror and both forge
  rename ports via `npm run sync:editions` (byte-/rename-identical — no hand-porting). The
  `kaola-workflow-adaptive-schema.js` byte-group was untouched (the grammar is fully expressible in the
  validator; R1/R2 consume only pre-existing schema exports).
- Tests (Claude-only surfaces, RED-first): `test-adaptive-node.js` pins the R1 co-run (mixed
  read+leg-write running set), the G4 merge-fence held under co-run then released after drain, each of
  the four fail-closed rows byte-identical to main, the mixed-set crash-reconcile, and the R2b band-cap
  matrix; `test-validation-allowband.js` pins the `isScratchObservableSurface` predicate (admits
  `docs/decisions/**` + non-test-consumed `docs/**`, rejects `docs/api.md`/`CHANGELOG.md`/`README.md`);
  `test-adaptive-handoff.js` pins the R2a freeze refusals end-to-end. All four
  `npm run test:kaola-workflow:{claude,codex,gitlab,gitea}` chains verified green (serial, unwaived).
- **Two known fenced residuals, deferred to #644** — both surfaced by this run's own `code-reviewer`
  and independently confirmed real-but-fenced by its `adversarial-verifier` (each reproduced
  end-to-end); neither is a verdict-time invariant violation and neither weakens the shipped fail-closed
  matrix, so both are non-blocking follow-ups:
  - **A1 (medium):** a live `kind:'gate'` (main-session-gate) is invisible to the co-open
    preconditions and to the G4 merge fence (`liveReadsAtMerge` counts `kind==='read'` only), so a
    `{main-session-gate ∥ writer}` antichain can R1-co-open behind a live gate — a window that is new
    in this change. It is fenced by single-orchestrator discipline (the gate's observation window is
    the orchestrator's own work window; no documented `orient`/resume path issues `open-ready` inside
    it), by the write-set-aware G3 (a production-set writer cannot antichain a gate at freeze, so only
    docs-band bytes can land), and by atomic close (merged bytes always belong to a node closed in the
    same `close-node`). The follow-up counts `kind:'gate'` at the merge fence and holds the relaxed
    co-opens behind a live gate.
  - **A2 (low):** the plan's `validation_test_consumes` widening (`testConsumedExtra`) is not threaded
    into `tryR2bLeglessCoopen`'s `scratchObservableWriteSet` call, so a self-hosting fork's extra
    test-consumed prose is admitted to the runtime R2b band. Fork-only, bounded to a spurious refute by
    the scratch contract (no post-domination ⇒ no laundering channel). The follow-up is a one-arg
    thread-through.

## Non-goals

- Does not touch the routing-surface (#400 six-surface) prose or the planner agent profile — the R2
  observation contract is composed inside `adaptive-node.js` when it opens the annotated node, and no
  user-facing routing prose describes the read∥write hold today (matching the #622 co-open precedent,
  which touched no routing surfaces). Teaching the planner to *author* `observes: scratch` is a
  separate follow-up; the freeze-time predicate already guides it (a bad annotation refuses).
- Does not relax any of the other 17 audited guards, and does not add a new agent or role.
