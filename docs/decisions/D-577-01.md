# D-577-01. Track the opencode hook-adapter plugin under `templates/opencode/plugins/` (canonical source + regeneration path)

Date: 2026-06-30
Status: Accepted
Issue: #577
Related: D-530-02 (opencode stays additive — runtime-not-forge; no #307 four-chain obligation)

## Context

The opencode edition ships a hook adapter plugin at `.opencode/plugins/kaola-workflow-hooks.js`
that bridges Claude-style JSON hook payloads into the runtime-neutral shell scripts shared across
all editions. Prior to this decision, that file had **no tracked canonical source**: it existed
only as an untracked, gitignored artifact under `.opencode/` (gitignored by `.opencode/.gitignore`).

This caused two concrete failures when working in a clean git worktree (fresh clone or a `git
worktree add`):

1. **`install-opencode.sh` silently deployed nothing.** The installer's plugin step used
   `.opencode/plugins/kaola-workflow-hooks.js` as its own source and swallowed failure with
   `2>/dev/null || true`, so a missing file produced no error and the deployed project contained
   no hooks plugin.
2. **`test-opencode-edition.js` (A11, P1, G1, H1) failed.** The hook-adapter assertions and the
   install-simulation assertions that depend on the plugin could not pass because the plugin was
   absent in a clean worktree.

The root cause was that the sync script (`sync-opencode-edition.js`) handled agents, commands, and
hooks (`.sh`) as generated outputs from canonical sources, but had no step for the plugin — it was
treated as hand-authored and expected to be present as an untracked file, a fragile assumption that
broke on every clean checkout.

## Decision

1. **Add a tracked canonical source** at `templates/opencode/plugins/kaola-workflow-hooks.js`,
   byte-identical to the live adapter, outside the gitignored `.opencode/` tree. This file is
   tracked in git and is the single authoritative source for the plugin.

2. **`sync-opencode-edition.js --write`** gains a `writePlugin()` step (mirrors `writeHooks()`):
   it byte-copies `templates/opencode/plugins/*.js` → `.opencode/plugins/`. `--check` asserts
   parity: a missing or drifted `.opencode/plugins/kaola-workflow-hooks.js` is a parity failure,
   not a silent pass.

3. **`install-opencode.sh`** deploys the plugin from `templates/opencode/plugins/` (the tracked
   canonical source), never from `.opencode/plugins/` itself (which would be a self-referential
   copy). The silent `2>/dev/null || true` suppression is removed; a missing tracked source is a
   loud, fatal install error.

4. **`test-opencode-edition.js`** self-provisions via `sync --write` before its assertions, and
   adds an `A11-canon` assertion: the tracked source exists AND the regenerated plugin is
   byte-identical to it. This assertion closes the gap permanently — if the tracked source ever
   drifts from the deployed plugin, parity fails loudly rather than silently.

## Scope

This is an opencode-edition-only change (additive, per D-530-02). The opencode edition is not
wired into `npm test`, `edition-sync.js`, `install.sh`, or the SIX routing surfaces. No #307
four-chain obligation applies. Validation runs via `node scripts/test-opencode-edition.js`.

## Consequences

- A fresh `git clone` followed by `node scripts/test-opencode-edition.js` (or
  `./install-opencode.sh`) passes without manual seeding of `.opencode/plugins/`.
- The gap cannot silently reopen: A11-canon fails loudly if the tracked source is absent or
  diverges from the deployed plugin.
- `install-opencode.sh` fails loudly (not silently) if the tracked plugin source is missing.
- The `.opencode/plugins/` directory remains gitignored; its contents are always regenerated
  from `templates/opencode/plugins/` on demand, never tracked directly.

## Alternatives considered

- **Keep the plugin as a hand-authored untracked file; seed it from `main` before running the
  test.** Rejected: this is manual toil on every clean worktree and does not fix the install
  regression. It also means the plugin has no tracked home and cannot be reviewed or diffed.
- **Remove the `.opencode/` gitignore exception for the plugin only.** Rejected: selectively
  unignoring `.opencode/plugins/kaola-workflow-hooks.js` while leaving the rest of `.opencode/`
  gitignored is fragile and inconsistent. The canonical-source-in-`templates/` model is already
  used for agents, commands, and hooks; extending it to the plugin is the consistent choice.
