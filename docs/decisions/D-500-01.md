## Context

Issue #500 is "a decision, not a build": three advertised parallel-write makespan levers are inert on
the live path. The ask is to decide — **per lever** — between **WIRE** (make reachable on the live
path) and **RELABEL** (mark experimental/deferred + fix docs so the "shipped" status is honest).
Principle: a shipped capability should be either reachable on the live path or honestly labeled —
never advertised-but-dormant. "Cheapest sufficient, pick one per lever"; a mix is allowed.

## Recommendation (per lever — PENDING APPROVAL)

### L1 — `write_overlap_policy` relaxation → **GUARDED-WIRE as a follow-up** (else near-free RELABEL floor)

The makespan win is real (shared-infra siblings serial-degrade today; relaxation co-opens them), but
"cheap AND safe" is refuted: the cheap consent-forward at `adaptive-node.js:3351` decouples
lane-formation from leg-provisioning and corrupts when `KAOLA_LEG_ISOLATION=0`; the safe leg-coupled
wire is not cheap and the shared-infra-coarse path has never run end-to-end. **Do not do the cheap
wire.** If the owner wants the shared-infra co-open win, build the *guarded* wire (leg-couple at
`:3816` + #400 6-surface activation prose + a new shared-infra-coarse end-to-end test). Otherwise the
RELABEL floor is nearly free: L1 is already labeled "#463 OPEN / step 1 / default-off"; only a
warning that the live caller never forwards consent (so `coarse` silently serial-degrades) is missing.

- **Flip-premise:** the owner's makespan valuation of the shared-infra co-open shape — *no live plan
  authors it today.* Common/valuable enough → WIRE; narrow win not worth the leg-couple + prose +
  new-test burden → RELABEL.
- **This is the only genuine owner value-call of the three.**

### L2 — `KAOLA_LEG_ISOLATION` → **WIRE**

The issue's "DORMANT-by-design" premise is **factually stale**: #463 slices 2–6 shipped, the engine is
COMPLETE (CHANGELOG "Closes #463"), AC18 live probe PASS. RELABEL-toward-dormant would make the docs
*less* honest, contradicting the shipped state — the wrong direction by the issue's own principle.
WIRE = document the **complete activation recipe** (`KAOLA_LANE_CONTAINMENT` + `KAOLA_LEG_ISOLATION` +
`--write-overlap-consent`) across the 6 plan-run surfaces and fix the stale "DORMANT" code comments at
`adaptive-node.js:3366-3370` / `:3890-3897`. Naming only `KAOLA_LEG_ISOLATION` without the full recipe
would be an *incomplete* activation doc.

- **Flip-premise (factually false today):** undiscovered leg-routing incompleteness (provisioning-only).
  n2 + AC18 PASS + the `SYNTH-DISJOINT-END-TO-END` test found the opposite. Only flips on out-of-band
  evidence of incompleteness this read-only run did not surface.

### L3 — `speculative_open_policy:consent` → **WIRE the prose** (+ file planner-rubric separately)

Fully runtime-wired ×4 editions, all tests green, least risky (read-only, evidence-discard rollback,
no env-toggle). The prose omission was a known, acknowledged deferral (CHANGELOG). WIRE the prose so the
consent path is reachable + honestly documented (1 `docs/plan-run-cards/speculative-open.md` card + six
`<!-- CARD: speculative-open -->` markers + a driver line). The *realizable* makespan win additionally
needs a planner rubric teaching the post-gate read-only topology — **file that as a separate follow-up**;
do not let the prose wire imply the proactive win lands without it.

- **Flip-premise:** reachability bar → WIRE; realizable-win-without-a-rubric bar → RELABEL as
  "functional-but-unexercised until a planner rubric ships."
