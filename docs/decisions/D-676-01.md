# D-676-01 — Archive-completeness gate is source-relative, not an absolute per-path floor

- **Status:** accepted
- **Date:** 2026-07-13
- **Context:** #676 (archive-completeness verification before live-copy deletion); bundle-675-676 run.

## Decision

`verifyArchiveComplete` verifies that the archive DEST preserves **every evidence file that actually exists in the live SOURCE** (source-relative), rather than enforcing an **absolute per-`workflow_path` floor** (adaptive ⇒ must contain plan + state + finalization-summary + node-evidence; fast/full ⇒ state + summary) as issue #676 literally proposed.

The source set is the top-level `{workflow-plan.md, workflow-state.md, finalization-summary.md, fast-summary.md}` plus every `.cache/*.md` file **except** the fixed-name machinery sidecars `{final-validation.md, run-gaps-manual.md, selection-evidence.md, doc-docking.md, doc-updater.md}`. `workflow-state.md` is retained as a single unconditional archive-identity anchor. Only the copy+verify (linked-run) path is gated; the atomic in-place `renameSync` cannot drop files. An incomplete archive returns a typed `archive_incomplete` refusal before any live-copy deletion or roadmap/issue/label side effect.

## Why (operator value call, routed to the consent valve)

Both contracts prevent the target incident (a run whose live folder held full evidence but whose archive received only `sink-receipt.json`, then deleted both live copies — vrpai-cli#885). The source-relative gate is **strictly more precise**: it refuses exactly when the archive LOST evidence the source had, catching that incident while:

- **breaking zero existing test fixtures** — the absolute floor broke 37+ fixtures (`test-bundle-finalize.js` ~35, `test-claim-hardening.js` 2, the two forge `test-*-workflow-scripts.js`) that finalize minimal/adaptive projects without seeding a summary; and
- **preserving the shipped minimal-finalize contract** (`#522(C)`: "finalize with no `workflow-plan.md` must succeed") — a minimal source requires only what it has.

The absolute floor is a proxy for "the source produced this evidence"; a real run always meets it, so the two agree on real runs, but the floor over-fires on minimal sources. Because `copyDir` is fully recursive, a faithful archive always carries every source file, so requiring "dest ⊇ source evidence" never false-refuses a good archive (fail-closed-safe over-inclusion).

The value call — which archive-completeness contract to ship, given the 37-fixture blast radius and the shipped minimal-finalize contract — was routed to the operator (machines decide facts; humans decide values) and resolved in favor of source-relative.

## Consequences

- An archive that drops any source-held node evidence (free-form or role-named `.cache/*.md`) is refused before deletion; both live copies survive (`archive_incomplete`).
- The node-evidence enumeration is name-shape-agnostic (any `sanitizeNodeId` id), closing the adversary-found glob gap where free-form ids (`design.md`, `review.md`) and fast/full role-named evidence (`planner.md`, `code-reviewer.md`) slipped a `/^n\d*-.+\.md$/` glob.
- Residual (deferred): a plan node id colliding exactly with a reserved sidecar name would be denylisted — theoretical, no shipped role produces those names, strictly better than the pre-fix glob.
