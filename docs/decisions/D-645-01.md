# D-645-01. A canonical root axiom file reached by embedding, not per-edition copies, with a byte-identity drift guard

Date: 2026-07-09
Status: Accepted
Issue: #645
Related: #472/#463/#439/#486 (Maximize Workflow Efficiency by Faithful Decomposition — the axiom
         layer supplies the tie-breaking order those topic-specific rules already presuppose);
         #501 (Self-Sufficient by Default — axiom 5, "own your own verdicts", restates it at
         Layer 0); #538 (Adaptive Is the Default — axiom 3, "cheapest sufficient mechanism",
         restates it); #400 (six-surface routing-prose rule — the axiom pointer on the `next`
         surfaces rides the same `required-blocks.js` manifest); #643/D-643-01 (evidence-contract
         registry — the sibling "one canonical source, verified everywhere" pattern this decision

## Context

Several precedence rules already existed in this repo, each independently justified for its own
topic: decomposition width and scheduling order, CI/CD never being a required gate, adaptive
running unconditionally by default. None of them named a single, cross-cutting tie-breaking order
an agent could fall back on when a situation is not already resolved by a specific rule, gate, or
refusal — only ad hoc judgment. #645 asked for that missing Layer 0: a small, stable set of axioms
in priority order, cheap enough to read on every dispatch, that never contradicts an existing
gate and never invents a new one.

## Decision

1. **One canonical file, `templates/axioms.md`, not per-edition copies.** `templates/` carries no
   runtime `require()` consumer (unlike `kaola-workflow-adaptive-schema.js`, which the four editions
   all `require()` and which is therefore kept in sync via `BYTE_IDENTICAL_GROUPS`). A physical
   per-edition copy of the axiom text would need the same byte-identical-group machinery for no
   runtime benefit; embedding the canonical text into surfaces that are already byte-locked for
   other reasons is the cheaper mechanism.
2. **Reach: embed, don't copy.** The identical `## First Principles` block is embedded byte-for-byte
   into all six workflow-init CLAUDE.md-template surfaces (3 Claude commands + 3 Codex SKILL packs),
   at one fixed anchor — immediately after the last `## Non-Negotiable Rules` bullet and immediately
   before `## Kaola-Workflow`. The drift guard is a new `simulate-workflow-walkthrough.js` scenario,
   `testAxiomBlockByteIdentity`, which asserts `templates/axioms.md` opens with `## First Principles`
   (a blanked-file false-green guard) and that its trimmed content is byte-identical inside each of
   the six embeds. This is a content-comparison guard, not a `BYTE_IDENTICAL_GROUPS` entry — that
   mechanism is for N>1 copies of the same logical *file*, not a canonical source embedded into a
   larger consumer document.
3. **A reference pointer, not the full block, on the six `next` routing surfaces.** The axiom
   layer's tie-breaker and tighten-only rules are also relevant at plan-run dispatch time, so a
   short, provenance-free pointer paragraph (not the whole five-axiom block) is required on all six
   generated `next` surfaces, added as a new `nx-first-principles` entry (`topic:'next'`,
   `runtime_tag:'both'`, `surface_type_tag:'both'`) in `templates/routing/required-blocks.js`'s
   single-source manifest. The obligated surface set is *derived* from the tag pair by
   `scripts/test-route-reachability.js`, never a hand-typed file list — the same mechanism #630
   built to make a 4-of-6 propagation gap structurally impossible.
4. **Tie-breaker protocol is optional and non-blocking.** When no shipped rule covers a situation,
   walk the axioms in priority order and record a one-line derivation in the node's `.cache`
   evidence. This derivation is deliberately *not* wired into `checkEvidenceShape` or any other
   close-time completeness check — it is a reasoning trace for a human or a later agent to read,
   not a proof obligation, and its absence never blocks a gate.
5. **Tighten-only is the hard, safety-relevant boundary.** An axiom may only make an agent stricter,
   never looser. The axiom text itself states this: never cite an axiom to justify skipping a typed
   gate, refusal, or barrier — gates define the allowed space, axioms only break ties inside it. This
   forecloses the failure mode where a plausible-sounding "efficiency" or "human-time" appeal is used
   to talk an agent out of an existing correctness check.

## Consequences

- No scheduler, validator, or gate *logic* changed anywhere in the codebase — every touched `.js`
  file gained either an additive assertion (the drift guard, the `required-blocks.js` manifest
  entry) or new prose; the adaptive scheduler, plan-validator grammar, and existing gate roles are
  byte-unchanged.
- The axiom file and its embeds are provenance-free (no `#NNN`, `D-NNN-NN`, `INV-NN`, or `ADR`
  tokens), per the existing prompt-surface provenance ban.
- Future divergence between `templates/axioms.md` and any of the six embeds, or a missing axiom
  pointer on any of the six `next` surfaces, reds `npm test` immediately rather than drifting
  silently.

## Non-goals

- Does not replace or supersede any existing topic-specific precedence rule (decomposition width,
  CI/CD-not-a-gate, adaptive-default, etc.) — axioms apply only when no more specific rule, gate, or
  refusal already resolves the situation.
- Does not add a new required evidence token or close-gate check for the tie-breaker derivation — it
  stays optional by design.
- Does not change the consent-halt valve's mechanics — axiom 4 restates the existing
  values-vs-facts routing principle, it does not re-implement or relocate it.
