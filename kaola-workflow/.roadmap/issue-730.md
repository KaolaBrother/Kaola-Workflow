THE THIRD REJECTION AND ITS ROOT CAUSE — worth carrying because it nearly shipped twice: the ownership partition was built on unresolvedInScopeFixes, which is the GATE predicate. It requires EXPLICIT scope==='in_scope' && action==='fix' and is deliberately fail-OPEN so that omission cannot manufacture a blocker. Repair asks the OPPOSITE question and must fail CLOSED. A schema-1 flat row such as 'finding: id=F-A status=open severity=high file=scripts/a.js' carries neither scope nor action, so the gate set excluded it and the brief silently truncated to a single-writer assignment — the exact defect the issue forbids. Fixed by adding repairResponsibleFindings to adaptive-schema.js: exclusion by POSITIVE PROOF of non-blocking only (explicitly resolved/deferred status, explicitly non-in_scope scope, or explicitly non-fix action); a row missing those fields is INCLUDED. Mutation-proven — reverting to the gate predicate reds 6 assertions, with the payload showing consumed_by='impl-b' and assigned_uids=['F-B'] while F-A, owned by impl-a, sits in context_findings. LESSON TO APPLY TO EVERY REMAINING CHILD: state at each predicate which question it answers and which direction it fails, and pin BOTH directions with a case that OMITS the optional fields entirely.
